Wunderite

Agency: Northeastern Scout
Role: UX Designer, Visual Designer
Software Used: Sketch, Invision

Overview

This project was done with a team at Northeastern's student led design studio: Scout! In this project my team helped Wunderite, an insurance management software, with branding and UX design.



Guiding Questions

Before starting with any sort of design, we wanted to make sure we knew what the client wanted or expected. This included questions like:

  • What makes Wunderite different from other software?
  • Who would be using Wunderite? Is it a consumer facing software or is it for businesses? Could it be used by both?
  • Why would someone need to use this software?



Outcomes

The final deliverable for the client was a full site redesign with designs given as Sketch files, a site developed with a front and backend, and a brand book for future print materials.

User Personas and Research

Before diving into wireframes and designs, we decided to focus on who the users of the platform would be. We came up with a list of users that the software could possibly be for and presented it to the client.

Wireframing

We had two sets of wireframes, with two different types each. We built a set of wireframes that would be insurance broker facing and one that would be insurance client facing. For each user type, we made a set of lofi and hifi wireframes.

In making the lofi wireframes, we would regularly consult with the client about what a broker would want in platform like this. What would be convenient to have or that this broker would use regularly? In these wireframes, we wanted to make sure the functionality was there before the actual designs.

So for example, a lofi wireframe would look like this: (click image to expand)

In this wireframe, we're showing what a broker would see on first login. We wanted to make sure that the broker would see the most important information at the top. This would include the clients that have deadlines coming up soon.

Next, the user would have a few actions they would use regularly, like see all of their clients, create a new risk profile, or file a new claim for their client.

And at the bottom of the dashboard, the broker would be able to find clients that have they have worked on recently. That way, the broker wouldn't have to go through the entire list of of clients every time.

After wireframing all the functionality in the site, we fleshed out designs for each lofi wireframe. In this stage, we gave a branding voice and color to the site.

Summary

There were many, many more wireframes and designs that were made for this client. The project was a great exercise in UX design and things to consider when redesigning a site. On top of learning more about UX, it was really informative learning about business insurance, in terms of how it differs from personal insurance and how insurance brokers and clients communicate with each other.
